## 2. 화면 구성

- **스플래시**: 2초 후 홈으로 이동 (종교·국가 선택 없음)
- **홈**: 랭킹 탭(오늘/이번주/이번달/1년, 종교별/계정별/국가별), 종교별 응원 포인트 그리드, **활동 소식** 섹션, **게시판** 섹션(최근 5개 제목 + 전체보기), 하단 네비(홈/응원하기/활동 소식)
- **계정(사람 아이콘)**: 테스트 유저 A·B·C 선택, 계정 이름 입력, 나의 종교·국가 선택/재선택, Google 로그인
- **응원하기**: 금액 선택(1/10/100/1000/기타) 후 **결제** 버튼으로 포인트 적립
- **활동 소식**: 활동 소식 전용 목록(이미지+요약 3개), 상세 화면 (/board, /board/:id)
- **게시판**: 활동 소식과 별도. 메인에는 최근 5개 제목만 노출, 전체보기 → 게시판 전체 10개 목록·상세 (/board-general, /board-general/:id)
- **약관**: 비영리 아님, 기부금 영수증 대상 아님, 수익 일부 사회공헌 사용 가능 문구

## 12. 2026년 3월 16일 업데이트

### 12-1. 빌드·푸시·배포
- 로컬 `flutter build web` 후 `build/web` 커밋 → GitHub main 푸시 → Vercel 자동 배포
- Firebase CLI: 프로젝트 전환(fastapi-js → kindly-god), `.firebaserc` default: kindly-god
- Firestore 보안 규칙: `npx firebase use kindly-god` 후 `npx firebase deploy --only firestore:rules` 로 kindly-god 프로젝트에 배포 완료

### 12-2. Vercel 배포 후 channel-error (Firebase 연동 실패)
- 증상: 로컬은 정상, 배포 사이트에서 랭킹 무한 로딩, 아이디 중복 검사 무반응
- 콘솔 에러: `PlatformException(channel-error, Unable to establish connection on channel., null, null)` (Auth/Firestore JS 채널 미연결)
- 원인: Flutter 웹 PWA 서비스워커가 예전 빌드(main.dart.js 등)를 캐시해 새 배포가 반영되지 않음
- 확인: Firebase Auth 승인된 도메인에 kindly-god.vercel.app 이미 등록됨 → 도메인 문제 아님
- 해결(당시): 시크릿 모드 또는 브라우저에서 해당 사이트 서비스워커 Unregister + Clear site data 후 재접속 시 정상

### 12-3. PWA 서비스워커 비활성화 (캐시 문제 근본 해결)
- 웹 빌드 시 `--pwa-strategy=none` 적용 → 서비스워커 미생성
- 명령: `flutter build web --release --pwa-strategy=none`
- 효과: 배포 시 항상 최신 JS 로드, 사용자에게 캐시 삭제 요청 불필요. 기존 방문자도 1~2회 재접속 시 최신 버전으로 전환

### 12-4. 빌드·푸시·배포 재실행
- `flutter clean` → `flutter pub get` → `flutter build web --release --pwa-strategy=none`
- `git add -f build/web` → commit → push origin main (Vercel 자동 배포)

### 12-5. 천국의 땅 분양 (가칭) — 삭제함, 나중에 적용용 코딩만 보관

### 13-7. 계정 탭에 국가·종교 표시
- 계정 랭킹 리스트에서 아이디(표시명) 아래에 **국가 · 종교**를 `subtitle`로 표시
- 서버 계정: `UserProfile.religionId`/`countryId` → `defaultReligions`/`defaultCountries`에서 이름 매핑
- 로컬 테스트 계정(A/B/C): 종교/국가 정보가 없으므로 미표시
